Emergency Procedure

  2. Do not touch the rotating shaft.
  3. Stop MATLAB execution.
  4. Notify the instructor or TA.
  5. Do not restore power until authorized.

Common Problems

  • No telemetry: check power, COM port, CAN wiring, and device ID.
  • Velocity has the wrong sign: stop and correct the sign convention.
  • Command saturates: lower the gain or reference.
  • Telemetry age rises: stop the experiment and inspect communication.

Lab Overview and Readiness

In this lab, MATLAB performs the controller calculation. The SPARK MAX is used as the motor power stage, encoder interface, and safety/watchdog device.

Learning Outcomes

  1. Implement P, PI, and PID speed control in MATLAB.
  2. Measure rise time, overshoot, and steady-state error.
  3. Recognize saturation and integral windup.
  4. Apply basic anti-windup and derivative filtering.
  5. Relate sample timing and delay to stability.

Checkpoint 2

Proportional Speed Control

Time: 25 min

Control Logic

The controller compares desired speed with measured speed. Proportional action creates a control effort that grows with the error.

error = reference − measured velocity
P = Kp × error

% under "TODO STUDENT 1 — PROPORTIONAL ACTION":

P = Kp * controlError;

% Under "TODO STUDENT 4 — COMPLETE THE CONTROLLER EQUATION":

Checkpoint 3

Integral Action and Anti-Windup

Time: 25 min

Integral Action

Integral action accumulates error over time. Persistent error therefore produces an increasing corrective action.

Windup

If the actuator is saturated while error continues to accumulate, the integral state can become excessively large and delay recovery. 

% Under "TODO STUDENT 3 — CANDIDATE INTEGRAL STATE":

candidateIntegral = integralState + controlError * dt;
candidateIntegral = clamp( ...
    candidateIntegral, -INTEGRAL_LIMIT, INTEGRAL_LIMIT);

% Under case "PI":

I = Ki * candidateIntegral;
D = 0.0;

% Under "TODO STUDENT 4":

controlUnsaturated = P + I;

Add anti-windup under: TODO STUDENT 5 — BASIC ANTI-WINDUP 

% One acceptable strategy:
% integrate when the output is not saturated,
% or when the error would drive the command back toward the valid range.

notSaturated = abs(controlUnsaturated) <= MAX_DUTY;

drivesBackFromUpperLimit = ...
    controlUnsaturated > MAX_DUTY && controlError < 0;

drivesBackFromLowerLimit = ...
    controlUnsaturated < -MAX_DUTY && controlError > 0;

if notSaturated || drivesBackFromUpperLimit || drivesBackFromLowerLimit
    integralState = candidateIntegral;
end

Checkpoint 4

Derivative Action and Filtering

Time: 20 min

Derivative Action

Derivative action responds to how quickly the measured speed is changing. It can improve damping, but it also magnifies measurement noise. 

% Under "TODO STUDENT 2 — DERIVATIVE OF THE MEASURED SPEED":

rawMeasurementDerivative = ...
    (measuredVelocity - previousVelocity) / dt;

filteredMeasurementDerivative = ...
    DERIVATIVE_FILTER_ALPHA * rawMeasurementDerivative + ...
    (1 - DERIVATIVE_FILTER_ALPHA) * ...
    filteredMeasurementDerivative;

D = -Kd * filteredMeasurementDerivative;

% Under case {"PID", "PID_DELAY"}:

I = Ki * candidateIntegral;

% Under "TODO STUDENT 4":

controlUnsaturated = P + I + D;

Lower α

More smoothing, less noise, but slower derivative response.

Higher α

Faster derivative response, but more sensitivity to noise.
